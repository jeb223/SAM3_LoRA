# 面向井下图像分割的参数高效多尺度语义增强 SAM3 方法

## 摘要

井下视觉场景具有低照度与非均匀照明并存、粉尘和噪声干扰显著、目标尺度差异大、细长结构与弱边界目标占比高、设备及人员等类别在复杂背景中易混淆等特点。直接应用通用域预训练的 Segment Anything Model 3（SAM3）时，模型可能难以充分适应矿井域的纹理退化、边缘弱化和语义混杂问题，同时全参数微调又带来较大的训练与存储代价。为此，当前仓库实现了一种面向井下图像分割的 SAM3 域适配框架。该框架在冻结大部分预训练参数的基础上，以低秩适配（Low-Rank Adaptation, LoRA）完成视觉与检测/分割关联模块的参数高效迁移；在像素解码路径中引入可训练的逐级残差融合适配器与浅层引导残差融合模块（Shallow-guided Residual Fusion Lite, SRF-Lite），增强细节纹理和多尺度上下文的联合建模；利用 SAM3 原有的语义输出分支增加语义级辅助监督，使实例掩码学习同时接受类别区域完整性约束；并扩展多类别文本查询组织方式与评估体系，以支持井下目标的类别感知分割评测。

需要严格说明的是，当前主训练脚本中实际生效的训练机制为 LoRA、残差多尺度/SRF-Lite 融合适配器、边界特征残差适配器及语义辅助损失。仓库中曾实现的低光照训练增强和显式边界 BCE/Dice 监督目前仅保留于历史脚本或配置字段中，未进入当前 `train_sam3_lora_native.py` 的优化总损失；自然语言指代表达数据已经构建，但当前 native 训练加载器尚未读取该表达文件。因此，下述“当前模型贡献”与“可扩展模块”作区分描述。

**关键词：** 井下图像分割；SAM3；低秩适配；多尺度特征融合；语义辅助监督；文本引导分割

## 1. 研究问题与方法动机

设井下图像为 \(I \in \mathbb{R}^{H \times W \times 3}\)，文本提示或类别查询为 \(q\)，目标是预测与查询对应的二值/类别掩码 \(M_q\)。与自然场景相比，井下图像分割面临如下主要困难。

1. **域偏移与样本有限。** SAM3 的预训练语义能力较强，但井下设备、支护结构、线缆、矿工及安全防护部件在纹理、尺度和照明分布上与通用数据存在明显差异；有限标注数据不适合对大模型进行高代价全参数微调。
2. **低照度及局部强光导致特征不稳定。** 矿灯形成的亮斑、暗部噪声和阴影遮挡会降低目标与背景之间的可分性，尤其影响小目标及细结构识别。
3. **尺度跨度与细节缺失。** 人员/大型设备需要较强全局语义，而电缆、管路、边缘部件等目标依赖浅层高分辨率信息。简单自顶向下特征相加难以充分选择和整合细节线索。
4. **弱边界和碎片化预测。** 黑暗背景、煤壁纹理及遮挡使目标轮廓模糊，细长物体容易断裂或与背景粘连。
5. **类别混淆与评估不足。** 同一图像中可同时出现多类设备与人员，仅依赖通用 `object` 查询或单一实例指标，难以反映类别语义区分和安全目标检出质量。

本文采用当前仓库的有效实现，将模型称为 **SAM3-UGSeg**（SAM3 for Underground Segmentation）。其整体思路是：保留 SAM3 的视觉-语言提示能力和主体参数，通过少量域适配参数改善井下特征分布；在掩码预测前增强多尺度像素表征；以实例损失和语义区域损失共同约束模型；使用类别感知的文本查询与更完整的评估协议衡量改进效果。

## 2. 与原始 SAM3 的总体差异

原始代码路径中的 `PixelDecoder` 对逐层特征采用插值后直接相加，再经卷积与归一化生成像素特征：

\[
P_l^{\mathrm{base}} =
\sigma\!\left(\operatorname{GN}\left(\operatorname{Conv}
\left(F_l + \mathcal{U}(P_{l+1})\right)\right)\right),
\]

其中 \(F_l\) 表示当前尺度特征，\(\mathcal{U}(\cdot)\) 为上采样，\(P_{l+1}\) 为更深层特征。当前模型在此基础上增加了三类关键改变：

1. 以 LoRA 和轻量适配器替代全参数更新，使预训练 SAM3 能以较低训练成本适应井下域。
2. 将简单相加的像素解码路径扩展为“逐级可学习残差融合 + SRF-Lite 全金字塔浅层引导融合 + 边界特征残差增强”的解码结构。
3. 在实例检测与掩码损失之外，将原始 SAM3 已输出但当前基础训练路径未专门利用的 `semantic_seg` 结果接入 `SemanticSegCriterion`，形成区域级语义辅助监督。

此外，当前仓库扩展了类别查询组织、低照度推理预处理、语义指标与原分辨率框评测支持，为井下应用建立更匹配任务需求的数据与验证接口。

## 3. 当前模型方法

### 3.1 参数高效的井下域适配

为避免有限井下数据导致的大模型全参数过拟合与显存开销，模型首先冻结 SAM3 基础权重，在指定线性投影层引入 LoRA。对于预训练线性变换 \(W_0\)，适配后的前向传播为：

\[
y = W_0 x + \frac{\alpha}{r} BAx,
\]

其中 \(A \in \mathbb{R}^{r \times d_{\mathrm{in}}}\) 与
\(B \in \mathbb{R}^{d_{\mathrm{out}} \times r}\) 为可学习低秩矩阵，\(r\) 远小于通道维度。当前主配置设置 \(r=16\)、\(\alpha=32\)、dropout 为 0.05，仅选择注意力投影相关目标模块 `q_proj`、`k_proj`、`v_proj`、`out_proj`、`qkv` 与 `proj`，并配置对视觉编码器、DETR 编码/解码路径和分割相关模块进行适配，文本编码器及几何编码器保持冻结。

实现中，标准 `nn.MultiheadAttention` 被替换为可分别访问 Q/K/V/输出投影的 `MultiheadAttentionLoRA`，从而能够在注意力内部注入低秩更新。进一步地，代码在冻结基础网络后重新启用被标记为 `_sam3_trainable_adapter` 的融合与边界适配模块，并将所有可训练适配参数共同保存。这意味着当前检查点并非只含 LoRA 矩阵，而是包含 **LoRA + 像素解码轻量适配器** 的统一域适配参数。

**对应问题。** LoRA 保留通用视觉-语言先验并减少井下小样本训练的自由度，主要缓解域偏移、训练资源受限和全量微调过拟合风险；选择视觉与解码关联路径进行更新，有利于将适配容量集中于低照度纹理和目标定位/掩码形成过程。

### 3.2 逐级残差多尺度融合适配器

原始像素解码器仅对当前特征与上采样深层特征进行逐元素相加，缺少对两类信息关系的显式建模。当前模型在每一个上采样阶段插入 `MultiScaleFusionAdapter`。对于当前层 \(F_l\) 与上采样特征 \(\hat{P}_{l+1}=\mathcal{U}(P_{l+1})\)，其融合形式可表为：

\[
\tilde{P}_l = F_l + \hat{P}_{l+1}
+ \beta_l R_l([F_l \Vert \hat{P}_{l+1}]),
\]

其中 \([\cdot \Vert \cdot]\) 表示通道拼接，\(R_l\) 由 \(1 \times 1\) 卷积、GroupNorm、ReLU 和 \(3 \times 3\) 卷积组成，\(\beta_l\) 为可学习残差尺度，初始化为 0。零初始化使模块在初始阶段退化为原始 SAM3 的加法融合形式，从而避免随机适配器在训练开始时破坏预训练表示。

**对应问题。** 井下场景中，小型器件、细线缆和遮挡边缘依赖浅层细节，而矿工、液压支架等目标依赖深层语义。该适配器学习在每一尺度上补充“细节与语义交互后的残差”，比固定相加更适合恢复暗光条件下容易被深层压缩的信息，并降低细目标漏分和边缘过平滑风险。

### 3.3 SRF-Lite 浅层引导残差融合模块

为进一步整合跨尺度信息，当前配置启用 `SRFLiteFusion`，设置四级特征、瓶颈通道数 64、双线性对齐方式与残差系数初值 \(\alpha_{\mathrm{srf}}=0.1\)。设像素解码过程中形成的四级金字塔表征为 \(\{P_1,P_2,P_3,P_4\}\)，其中 \(P_1\) 对应目标高分辨率尺度。模块首先通过独立 \(1 \times 1\) Conv-GN-GELU 映射压缩各尺度特征，并上采样至共同空间分辨率：

\[
A_i = \mathcal{U}_i(\phi_i(P_i)), \quad A_i \in \mathbb{R}^{d_b \times h \times w}, \ d_b=64.
\]

随后由浅层高分辨率特征生成空间注意门控：

\[
G = \operatorname{Sigmoid}(\psi(A_1)).
\]

深层特征在与浅层细节位置一致处得到选择性强化：

\[
Z = [A_1 \Vert (G \odot A_2) \Vert (G \odot A_3) \Vert (G \odot A_4)].
\]

经卷积融合和归一化得到残差 \(\Delta_{\mathrm{srf}}\)，并回注到像素解码输出：

\[
P_{\mathrm{out}}' =
\operatorname{ReLU}\left(P_{\mathrm{out}}
+ \alpha_{\mathrm{srf}}\Delta_{\mathrm{srf}}\right).
\]

**对应问题。** 在井下图像中，暗区小目标和轮廓常仅在浅层保留局部梯度，而仅使用浅层纹理又容易将煤壁纹理、灯光反射误识别为目标。SRF-Lite 以浅层定位线索门控深层语义补充，使强语义响应优先落在具有细节证据的位置，面向“小目标/细长物体丢失”和“暗背景纹理干扰”两类问题提供针对性机制。

### 3.4 边界特征残差增强分支

当前分割头在像素表征与掩码预测器之间加入 `BoundaryRefinementAdapter`。模块使用两层 \(3 \times 3\) 卷积提取局部边缘敏感特征 \(E\)，并以可学习尺度 \(\rho\) 将其残差注入原像素特征：

\[
E = \varphi(P_{\mathrm{out}}'), \qquad
P_b = P_{\mathrm{out}}' + \rho E,
\]

同时生成边界预测输出：

\[
\hat{B} = \operatorname{Conv}_{1 \times 1}(E).
\]

增强后的 \(P_b\) 被送入实例掩码头和语义头，因此 `refine` 特征可通过掩码损失及语义损失获得梯度。

**实现边界与表述限制。** 当前主训练脚本不再计算 `pred_boundaries` 对应的边界标签 BCE/Dice 损失；`boundary_optimization` 字段仍存在于 YAML 中，但未在当前损失链路中读取。因此，在当前论文版本中应将该组件称为“边界特征残差增强分支”或“boundary-aware feature refinement”，不能宣称已经实施显式边界监督。若后续重新接入历史版本中的边界损失，该分支才可进一步表述为边界监督优化模块。

**对应问题。** 当前有效作用是利用局部卷积残差改善掩码形成前的高分辨率表征，针对弱边缘、粘连与细结构断裂提供结构性偏置；其独立边界预测能力仍需在接入边界监督并完成消融实验后验证。

### 3.5 实例-语义联合辅助监督

原始 SAM3 分割头本身包含 `semantic_seg_head` 输出。当前训练路径的关键变化不是新增该输出头，而是将其通过 `SemanticSegCriterion` 正式接入训练总损失。对于每个文本/类别查询，训练代码将对应实例掩码合并为语义区域真值 \(M^{\mathrm{sem}}\)，并对密集语义预测施加 focal 与 Dice 约束：

\[
\mathcal{L}_{\mathrm{sem}}
= \lambda_{\mathrm{sf}}\mathcal{L}_{\mathrm{focal}}
 \lambda_{\mathrm{sd}}\mathcal{L}_{\mathrm{dice}},
\]

其中当前配置为
\(\lambda_{\mathrm{sf}}=20\)、
\(\lambda_{\mathrm{sd}}=30\)、
\(\mathrm{focal\_alpha}=0.6\)、
\(\mathrm{focal\_gamma}=2.0\)，并将预测上采样至真值尺寸进行监督。配置设置 `presence_head: true` 以在无目标查询上避免不恰当传播掩码损失，同时设置 `presence_loss: false`，因此语义 presence 项在当前总损失中不产生有效优化贡献。

实例分割主损失仍包含框回归、分类/存在性和掩码项：

\[
\mathcal{L}_{\mathrm{inst}} =
5\mathcal{L}_{\mathrm{bbox}}
+2\mathcal{L}_{\mathrm{giou}}
+20\mathcal{L}_{\mathrm{ce}}
+20\mathcal{L}_{\mathrm{presence}}
+200\mathcal{L}_{\mathrm{mask\_focal}}
+10\mathcal{L}_{\mathrm{mask\_dice}},
\]

并保留 one-to-many 匹配输出的加权监督。当前优化目标可概括为：

\[
\mathcal{L}_{\mathrm{total}} =
\mathcal{L}_{\mathrm{inst}} + \mathcal{L}_{\mathrm{sem}}.
\]

由于基础参数及原有语义卷积头在 LoRA 初始化步骤中被冻结，语义损失主要通过可训练 LoRA 和新增融合/边界残差适配器反向塑造共享像素特征，这一设计保持了参数高效训练原则。

**对应问题。** 仅依赖实例匹配可能导致同类目标局部分裂、掩码区域不完整或背景噪声被错误划入局部实例。语义辅助目标强调查询类别对应的整体前景区域，有助于提高矿工、设备和结构部件等类别区域的一致性，并对类别混淆与实例碎片化形成补充约束。

### 3.6 类别感知文本查询训练

当前 `COCOSegmentDataset` 不再将一幅图中的全部目标统一归入通用 `object` 提示，而是依据 COCO `category_id` 将实例按类别名分组，并为每个类别构造独立 `FindQueryLoaded`：

\[
q_c = \text{category\_name}(c), \qquad
Y(q_c)=\{M_i \mid \operatorname{class}(M_i)=c\}.
\]

因此，同一井下图像中的矿工、安全帽、线缆、设备等可以分别接受其类别查询对应的监督，而不将不同类别对象错误混合到一个输出目标集合中。

**对应问题。** 井下安全感知不仅要求识别“某个前景”，还要求区分人员、防护装备、机械部件和风险物体。类别感知查询使 SAM3 的语言条件能力与标注类别对齐，为多类密集场景的语义判别提供直接监督基础。

### 3.7 推理端低照度可选预处理

当前推理脚本提供两种可选光照校正操作：gamma 变换和在 Lab 亮度通道上的 CLAHE 局部直方图增强。二者在模型输入之前执行，而可视化仍使用原始图像及原始尺寸。该设计允许在不修改训练权重的情况下分析暗区可见度改善对推理的影响。

**对应问题。** gamma 校正适合整体偏暗图像的亮度补偿，CLAHE 面向不均匀照明条件下局部对比度不足问题。由于当前 native 训练不包含对应随机低光增强，该策略应作为推理时可选预处理或消融因素报告，而不应写成已经完成训练-推理一致性鲁棒学习。

## 4. 数据资源扩展与当前接入状态

仓库已实现井下文本指代分割数据构建脚本。该模块基于 COCO 实例标注与图像内容，自动生成类别、位置、属性、特定描述和空间关系五类表达。例如，脚本依据掩码/框区域估计主颜色、相对尺度、方向、空间区域及邻近目标，再组合形成诸如 “the black cable” 或 “the support equipment in the lower left” 的表达，并记录对应 `target_ann_ids`。

现有统计文件显示：

| 数据拆分 | 图像数 | 实例标注数 | 类别数 | 自动构造的指代表达数 |
| --- | ---: | ---: | ---: | ---: |
| 当前井下训练拆分 `sam3_train_valid_phrases/train` | 2,383 | 25,900 | 15 | 315,877 |
| 当前井下验证拆分 `sam3_train_valid_phrases/valid` | 788 | 8,900 | 15 | 105,955 |
| 外部井下扩展 `external_400_100_phrases/train` | 2,400 | 4,444 | 21 | 60,331 |
| 外部井下扩展 `external_400_100_phrases/valid` | 600 | 1,171 | 21 | 15,703 |

该数据扩展可解决同类别多实例情况下“仅用类别名不能唯一指向目标”的问题，也可支持更真实的人机文本交互分割。然而，当前 `train_sam3_lora_native.py` 读取的是 `_annotations.coco.json` 并以类别名生成查询，尚未读取 `referring_annotations.json`。因此，指代表达数据集应在论文中表述为**已构建的数据资源与下一阶段训练接口**，除非后续接入加载器并重新训练/评测。

| 同类/异类实例混杂、区域不完整 | 仅实例匹配监督对类别整体区域约束不足 | `SemanticSegCriterion` 接入语义输出，focal + Dice 联合监督 | 当前训练生效 | 强化类别前景覆盖与区域一致性，抑制碎片化和语义误分 |
| 多类别安全目标需要按语义检索 | 通用提示不能直接对应不同井下类别 | 按 COCO 类别名生成独立文本查询 | 当前训练生效 | 保持类别-掩码对应关系，支持文本引导的多类分割 |
| 极暗或局部过曝输入 | 输入对比度低导致特征辨识困难 | gamma/CLAHE 推理预处理 | 推理可选 | 改善可见度；需以一致评测验证泛化效果 |
| 类别指标与安全召回刻画不足 | 单一 mask AP 不充分反映多类和空类误检 | APmask、APbbox、cgF1、语义 mIoU/F1、语义头指标、原分辨率 bbox 支持 | 当前验证路径生效（原分辨率 bbox 修改处于工作区） | 对实例质量、类别像素质量和负类误报进行更完整评价 |

## 6. 可归纳的学术创新点

### 创新点一：参数高效的井下域专用 SAM3 适配框架

本文并非对 SAM3 进行简单全量微调，而是构建了由注意力低秩更新与轻量像素解码适配器组成的联合适配机制。在保持预训练主干稳定的前提下，该方法使模型能够学习井下照明、纹理与目标形态的任务特异表征，并将训练产物组织为可独立加载的适配参数集合，适合矿山应用中多场景部署和模型快速切换。

### 创新点二：浅层引导的多尺度残差像素解码结构

本文针对井下目标“小、细、暗、边界弱”的联合特征，在 SAM3 像素解码器内同时采用逐级可学习残差融合与 SRF-Lite 全尺度增强。不同于原始固定加和方式，该结构通过残差初始化保留预训练基础能力，并利用浅层细节生成门控以选择性引入深层语义，从结构上增强暗光复杂背景下的空间定位和轮廓恢复能力。

### 创新点三：面向多类别井下目标的实例-语义联合适配监督

本文将类别感知文本查询与密集语义辅助损失结合：每类查询对应其实例集合，同时以合并区域监督语义输出，使域适配参数既学习单实例边界/定位，也学习类别区域完整性。该设计适用于矿工、防护装备、线缆和设备等同时出现且互相遮挡的井下场景。

### 创新点四：与矿山安全任务匹配的验证与文本数据扩展体系

当前实现将实例 AP、语义像素指标和 cgF1 纳入比较接口，并补充原图尺寸框评估，以降低单一分辨率或单一指标造成的结论偏差。同时，仓库已经构建属性与空间关系丰富的井下文本指代标注资源，为后续从类别条件分割扩展到指令式目标定位与分割奠定数据基础。

## 7. 实验设计与消融建议

为了使上述方法贡献在论文中具有可验证性，应以同一训练/验证拆分、同一阈值和同一后处理协议执行以下比较。仓库已有 `compare_sam3_metrics.py` 与 `validate_sam3_lora.py` 支持基础 SAM3 与适配模型的统一指标输出。

| 实验编号 | 配置 | 目的 |
| --- | --- | --- |
| B0 | 原始 SAM3，无适配权重，SRF 关闭 | 建立零样本基线 |
| B1 | SAM3 + LoRA | 验证参数高效域适配贡献 |
| B2 | B1 + `MultiScaleFusionAdapter` | 验证逐级多尺度残差融合贡献 |
| B3 | B2 + SRF-Lite | 验证浅层引导多尺度增强贡献 |
| B4 | B3 + 语义辅助损失 | 验证实例-语义联合监督贡献，即当前主模型 |
| E1 | B4 + 重新接入显式边界损失 | 验证边界监督能否进一步提升弱轮廓目标 |
| E2 | B4 + 重新接入低光训练增强 | 验证照明鲁棒性及训练-推理一致性 |
| E3 | B4 + 指代表达加载器与文本数据 | 验证从类别提示到细粒度自然语言提示的扩展效果 |

建议报告的评价指标如下：

1. `APmask`, `APmask@50`, `APmask@75`：评价实例掩码质量。
2. `APbbox` 与原分辨率 `APbbox`：评价目标定位性能，尤其适合只有框标注的外部数据。
3. `cgF1`, `cgF1@50`, `cgF1@75`：兼顾目标级召回及错误检出，适用于安全监测需求。
4. `semantic_mIoU`, `semantic_macro_F1`, `semantic_micro_F1`：评价多类别区域识别质量。
5. `semantic_head_mIoU/F1`：直接评估语义辅助分支是否真正学习到类别区域。
6. 模型可训练参数量、适配权重大小与显存占用：评价部署和训练成本。

当前仓库存在训练/验证 loss 记录，但未发现能够无歧义归属到全部当前模块组合且完成原始 SAM3 对照的最终指标文件。因此，论文中不应在未重新运行统一对照实验前写出“显著优于原始 SAM3”的定量结论；现阶段可准确表述为“提出并实现了上述机制，统一对照实验用于验证其效果”。

## 9. 论文表述中必须保留的工程事实

1. 当前主线中 `low_light_augmentation` 虽仍写在 `configs/my_lora_config.yaml`，但 `train_sam3_lora_native.py` 未构造或调用低光增强器；其完整实现仅可在 `history/train_低光照增强.py` 中找到。

3. `semantic_seg_head` 是 SAM3 分割头已有输出能力；当前创新是将 `SemanticSegCriterion` 接入 LoRA/适配器训练，而不是声称首次发明语义输出头。

5. 工作区中的 `validate_sam3_lora.py` 尚有未提交修改，新增了更严谨的原分辨率 bbox 评测路径；如用于正式论文实验，应固定该版本并保存运行配置、阈值、权重与结果 JSON。
